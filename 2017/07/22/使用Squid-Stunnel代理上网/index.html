使用Squid+Stunnel代理上网

作者:WangBinZhou    发布于: 2017-07-22

一、简介：

  作为一个IT从业者，经常需要到Google上面去查找一些技术相关的资料。但是由于……(此处省略若干字)，我们无法访问谷歌。因为上家单位属于互联网公司的缘故，我接触到了使用Squid + Stunnel科学上网的方式。
  关于下面的配置步骤，各位看官如果发现有错误或需要优化的地方，欢迎指正和发表建议！新手上路，请多多关照！

二、安装环境及原理：

  服务端：阿里云香港区ECS主机（CentOS 7.3.1611）一台；
  软件：squid-3.5.20-2.el7_3.3.x86_64 stunnel-4.56-6.el7.x86_64
  客户端：stunnel-5.39-win32-installer.exe & Proxy SwitchyOmega（Chrome扩展程序）
  原理：当客户端访问谷歌时,Stunnel的客户端通过和服务端建立起加密连接，由Squid代理用户去请求谷歌网站并把请求结果通过Stunnel的加密隧道返回给客户端；

三、服务端配置：
 1、服务端安装配置Squid：
#yum install -y squid                   //yum安装Squid
#vim /etc/squid/squid.conf              //修改配置文件
    # And finally deny all other access to this proxy
    #http_access deny all               //注释掉该行
    http_access allow all               //复制上一行，将“deny”改为“allow”
    # Squid normally listens to port 3128
    http_port 127.0.0.1:3128            //在端口前加入本地回环地址，只允许本地访问；
 2、服务端安装Stunnel：
#yum install -y stunnel     //yum安装Stunnel
 3、生成加密代理证书：
#cd /etc/stunnel
#openssl req -new -x509 -days 3650 -nodes -out stunnel.pem -keyout stunnel.pem     //生成证书
Generating a 2048 bit RSA private key
.....+++
....+++
writing new private key to 'stunnel.pem'
-----

into your certificate request.
What you are about to enter is what is called a Distinguished Name or a DN.
There are quite a few fields but you can leave some blank
For some fields there will be a default value,
If you enter '.', the field will be left blank.
-----
Country Name (2 letter code) [XX]:CN                            //国家字母代码
State or Province Name (full name) []:BeiJing                   //省份
Locality Name (eg, city) [Default City]:BeiJing                 //城市
Organization Name (eg, company) [Default Company Ltd]:          //组织名称
Organizational Unit Name (eg, section) []:                      //组织单位名称
Common Name (eg, your name or your server's hostname) []:       //当前系统的主机名
Email Address []:                                               //邮件地址

 结束后会在/etc/stunnel/目录下生成一个stunnel.pem的文件；

 4、配置Stunnel：
#vim /etc/stunnel/stunnel.conf      //stunnel.conf如果没有可以自行创建；
client = no
cert = /etc/stunnel/stunnel.pem     //配置加密代理证书
CAfile = /etc/stunnel/stunnel.pem
[https]
accept = 34567                      //服务端口
connect = 127.0.0.1:3128            //指向本地的Squid服务端口
 5、启动服务并检查：
#systemctl start squid.service      //启动Squid；
#stunnel        //启动Stunnel；
#ss -tunlp      //检测端口判断服务是否启动；
Netid State      Recv-Q Send-Q       Local Address:Port        Peer Address:Port
tcp   LISTEN     0      128              127.0.0.1:3128                   *:*           //squid服务            
tcp   LISTEN     0      128                      *:34567                  *:*           //Stunnel服务
 6、对公网开放34567端口；

 在Centos 7的默认防火墙Firewall上放行TCP:34567端口:

#firewall-cmd --zone=public --add-port=34567/tcp --permanent

 在阿里云控制台里面添加端口映射后，在客户端上面测试是否能够正常访问Stunnel端口：

C:\Users\wang>telnet Service_IP 34567

 如果命令执行后没有报错，cmd.exe窗口显示的是一个黑屏，说明服务能够正常使用；

四、客户端配置：
 1、安装stunnel.exe客户端后直接运行；
 2、配置Stunnel客户端：

  在电脑托盘里右键单击Stunnel图标，选择“Edit Configure”后清空预配添加如下配置:

client=yes
[https]
accept=127.0.0.1:8088       //在浏览器里面添加的代理地址和端口号；
connect=Service_IP:34567      //对应服务器的IP和端口；

  我使用的是Chrome浏览器，在里面添加了Proxy SwitchyOmega扩展程序，能够灵活的设置代理上网，也可以添加相应的匹配规则，让需要代理网址的走Stunnel访问，国内的网站正常（不走Stunnel）的访问；
