Linux常用命令总结-7、文本处理工具

2017-07-25
Linux常用命令总结

简介

  作为运维人员，我们经常需要查看、分析和统计日志等信息，在这里，我对初级的文本处理工具的学习做一个小总结。
本文提到的命令：
  查看文件内容： cat、more、less
  文件截取： head 和 tail
  按列抽取: cut
  文本数据统计： wc
  文本排序： sort
  去重复的行： uniq
  按关键字抽取：grep

文件查看命令

标准输入文件：cat

用法：

参数:
-E: 显示行结束符&；
-n: 对显示出的每一行进行编号；
-A: 显示所有控制符；
-b: 非空行编号；
-s: 压缩连续的空行成一行；

分页查看文件：more

用法：

参数：
-d: 显示翻页及退出提示；

按页查看文件：less

用法:

查看时常用的命令：
/文本: 搜索“文本”；
n/N: 跳到下一个或上一个匹配；

Note: man命令使用的就是less分页器；

显示文本前#行的内容：head

用法：

参数：
-c #: 指定获取前#字节；
-n #: 指定获取前#字节；
-#: 指定行数；

显示文本后#行的内容：tail

用法：

参数：
-c #: 指定获取后#字节；
-n #: 指定获取后#字节；
- #: 指定行数；
-f: 跟踪显示文件新追加的内容，常用日志监控；

按列抽取文本：cut

参数：
-d DELIMITER: 指明分隔符，默认tab；
-f FILEDS:
  #: 第#字段；
  #,#[,#]: 离散的多个字段，例如1,3,6；
  #-#: 1-3,7;
  -c: 按字符切割；
  --output-delimiter=STRING指明输出分隔符；
示例：

合并两个文件同行号的列到一行：paste

用法：

参数：
-d 分隔符: 指定分隔符，默认用TAB;
-s: 所有行合成一行显示；
示例：

文本数据统计：wc

  对文件或STDIN（标准输出）的数据，做单词总数、行总数、字节总数字符总数的统计。
用法：

参数：
-l: 只统计行数；
-w: 只统计单词总数；
-c: 只统计字节总数；
-m: 只统计字符总数；

文本排序： sort

  把整理过的文本显示在标准输出，不会改变原始文件。
用法：

参数：
-r: 执行反方向（由上至下）整理；
-n: 执行按数字大小整理；
-f: 选项忽略（fold）字符串中的字符大小写；
-u: 选项（独特，unique）删除输出中的重复行；
-t c: 选项使用c作为字符界定符；
-k X: 选项安装使用c字符分隔的X列来整理能够使用多次；

去除重复的行：uniq

  从输入中删除前后相接的重复的行；
用法：

参数：
-c: 显示每行重复出现的次数；
-d: 仅显示重复出现的行；
-u: 仅显示不曾重复的行；

Note: 连续且完全想用方为重复；
常和sort命令一起配合使用：sort userlist.txt | uniq -c

文件对比：diff 和 patch
比较文件：diff

  比较两个文件之间的区别；

注明第5行有区别（改变）。

复制对文件改变：patch

diff: 命令的输入被保存在一种叫做“补丁”的文件中。

使用-u选项来输出“统一的（unified）”diff格式文件，最适用于补丁文件。
patch: 复制在其它文件中进行的改变（要谨慎使用）
使用-b选项来自动备份改变了的文件；

文本过滤：grep

  文本搜索工具，根据用户指定的正则表达式字符及文本字符所编写的过滤条件，对目标文本逐行进行匹配检查，打印匹配到的行。
用法：

参数：
--color=auto: 对匹配到的文本着色显示；（Centos 7默认已使用alias设置此参数）
-v: 显示不被pattern匹配到的行；
-i: 忽略字符大小写；
-n: 显示匹配的行号；
-c: 统计匹配的行数；
-o: 仅显示匹配到的字符串；
-q: 静默模式，不输出任何信息;
-A # : after,同时显示匹配行的后#行;
-B #: before,同时显示匹配行的前#行；
-c #: context,同时显示匹配行的前后各#行；
-e: 实现多个选项间的逻辑or关系；

-w: 匹配整个单词；
-E: 使用ERE;
-F: 相当于fgrep，不支持正则表达式；

简介

  Linux命令分为内部命令和外部命令；
  内部命令： 由shell自带的命令。在系统启动时就调入内存，是常驻内存的，所以执行效率高。
  外部命令： 由系统的软件提供的命令；，用户需要时才从硬盘中读入内存；

hash缓存表：

  系统初始hash表为空，当外部命令执行时，默认会从PATH路径下寻找该命令，找到后会将这条命令的路径记录到hash表中，当再次使用该命令时， shell解释器首先会查看hash表，存在将执行之，如果不存在，将会去PATH路径下寻找。利用hash缓存表可大大提高命令的调用速率。
